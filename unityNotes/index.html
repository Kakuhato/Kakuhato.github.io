Unity学习笔记

碰撞

碰撞产生的条件是两个物体都有Collider，且至少有一个物体含有Rigidbody。Collider用于描述物体的边界，Rigidbody用于模拟物理效果。

Collision Detection（碰撞检测模式）

用于防止快速移动的对象穿过其他对象而不检测碰撞

Collider（碰撞器）

碰撞器类型

  • Box Collider（盒状）：

  • Capsule Collider（胶囊状）

  • Sphere Collider（球状）：仅3D

  • Mesh Collider（网状）：

    Convex：勾选后Unity会自动将网格凹凸不平的部分填平。如空心圆筒的碰撞箱会变为实心圆柱。

Is Trigger：若勾选，则该Collider失去物理效果，用于触发事件，进行没有物理效果的碰撞检测。如设计贯穿的攻击模式

刚体对象的子对象Collider会参与碰撞检测（父对象作为刚体整体，子对象描述边界）：复杂物体，父对象挂载Rigidbody但没有Collider，每个子对象挂载Collider但没有Rigidbody。通过子对象来描述父对象的边界。

Material（物理材质）

  • Dynamic Friction：移动摩擦力，描述物体停下来的阻力，为1时迅速停止
  • Static Friction：静止摩擦力，描述物体运动起来的阻力，为1时很难移动
  • Bounciness：弹力，为1时反弹不产生任何能量损失

碰撞检测函数

物理碰撞函数触发相关函数响应，在FixedUpdate之后，在Update之前。碰撞检测的响应频率与FixUpdate的频率有关。响应函数触发的条件是两个物体均含有碰撞箱（Collider）且至少一个物体含有刚体（Rigidbody）。
