碰撞

碰撞产生的条件是两个物体都有Collider，且至少有一个物体含有Rigidbody。Collider用于描述物体的边界，Rigidbody用于模拟物理效果。

Material（物理材质）

  • Dynamic Friction：移动摩擦力，描述物体停下来的阻力，为1时迅速停止
  • Static Friction：静止摩擦力，描述物体运动起来的阻力，为1时很难移动
  • Bounciness：弹力，为1时反弹不产生任何能量损失

2D物理

刚体

Simulated：勾选后，2D刚体以及所有子对象的Collider2D以及2D关节都能模拟物理效果

Sleeping Mode：对象处于静止状态时进入睡眠模式

  • Never Sleep：从不休眠，会一直进行检测计算
  • Start Awake：最初处于唤醒状态
  • Start Asleep：最初处于睡眠状态，但是可以被碰撞唤醒

Interpolate：物理更新间隔（FixedUpdate）之间的插值运算

  • Interpolate：根据前一帧的位置进行平滑处理
  • Extrapolate：根据后一帧的位置进行平滑处理

Kinematic：运动学类型，不受力的影响只能通过代码进行移动

Simulated：启用后充当一个无限质量的不可移动物体，和下面的参数一起勾选可以和所有刚体对象产生碰撞

Use Full Kinematic Contacts：如果不启用，则只能与Dynamic类型的刚体发生碰撞，不能与Kinematic类型以及Static类型的刚体发生碰撞

Static：只能与Dynamic类型的刚体碰撞，相当于无限质量的不可移动对象

碰撞器

Used by Composite：勾选后使用复合碰撞器

Auto Tiling：当Sprite Renderer的Draw Mode设置为Tiled模式时，勾选该选项，改变精灵尺寸大小，将自动更新碰撞器的尺寸来适配精灵尺寸

Composite Collider 2D：复合碰撞器，挂载在父对象上，子对象的碰撞器勾选Used by Composite，此时碰撞器合为一个异形碰撞器

Geometry Type：几何学类型，碰撞体顶点会组合成不同的几何体类型

  • Outlines：空心轮廓，边界碰撞器，其他碰撞体存在其中（相当于空盒子）
  • Polygons：实心多边形，其他碰撞体在其中会出现挤压（相当于实心盒子）

Vertex Distance：从复合碰撞器收集顶点时允许的最小间距值

Offset Distance：复合碰撞器相对单个碰撞器的偏移值，越大越“圆润”

Constant Force 2D（恒定力）组件

Forece：物理更新时施加在2D刚体上的线性力

Relative Force：物理更新时相对于刚体自身坐标系施加的线性力

Torque：扭矩力。2D模式下没有Relative Torque，因为使用的物理引擎为简化版的Box2D，3D使用的PhysX功能更完整。

效应器组件

区域效应器

需要搭配碰撞器使用，同时碰撞器应勾选Is Trigger和Used By Effector

Use Globle Angle：不勾选的话，施加力的角度，会以效应器挂载的对象的本地坐标系的X轴正方向为0度

Collider Mask：决定哪些层会受到效应器的作用

Force Angle：添加力的角度，X轴正方向为0度，Y轴正方向90度

Force Magnitude：添加力的大小

Force Variation：添加力的随机大小值，施加力时，会在Force Magnitude的基础上加减这个值的区间内进行选择

Force Target：区域效应器在目标游戏对象上施力的作用点

  • Rigidbody：目标点设置为刚体的当前质心
  • Collider：目标点设置为碰撞体的当前接触位置，可能产生扭矩

浮力效应器

Density：流体密度

Surface Level：定义浮力流体的表面位置，物体高于此线不会受到浮力作用

Flow Angle：流体流动方向

Flow Magnitude：流动强度，值越大，物体被推动的速度越快

点效应器

Distance Scale：效应器作用点到物体之间的距离缩放。计算距离时按照该值对距离进行计算

Force Source：力源是吸引或排斥目标对象的点，从该点定义相对于目标的距离。

Force Mode：

  • Constant：忽略力源与目标之间的距离
  • Inverse Liner：距离越远，力的大小呈线性减小
  • Inverse Squared：力的大小与距离呈平方反比

平台效应器

碰撞器只需要勾选Used By Effector，不需要勾选Is Trigger

Use One Way：是否使用单向碰撞行为，默认勾选

Use One Way Grouping：当平台由多个碰撞器组合时，通过该选项使所有碰撞器都设置为单向碰撞（对子对象的碰撞体有效）

Use Side Friction：是否在平台两侧启用摩擦

Use Side Bounce：是否在平台两侧启用弹性

Side Arc：左右两侧的响应弧度，通过碰撞法线判断

表面效应器

不需要勾选Is Trigger

Speed：表面维持的速度

Force Scale：在尝试达到表面速度时所施加冲击力的比例，实际施加的力 ≈ Force Scale × (目标速度 - 物体当前速度)，0表示禁用，但设置为1时，可能抵消加在物体上的其他力（如跳跃）因为此时效应器的行为类似于一个阻尼器，试图以“1:1的比例”修正物体速度，此时物体受到其他外力作用，效应器会“即时响应”并施加反向力来抵消速度变化。

Use Contact Force：是否对接触物体的表面施加力，如果选择会出现扭矩

四元数

欧拉角：空间中任意旋转，都可以分解成绕三个互相垂直轴的三个旋转角组成的序列

欧拉角旋转约定，heading-pitch-bank是最常用的一种序列约定，Y-X-Z约定。heading：物体绕Y轴旋转；pitch：物体绕X轴；bank：绕Z轴。这个序列意味着，Y轴旋转时，会带动X和Z轴一起旋转，但X轴的旋转并不能带动Y轴旋转，Z轴只能影响物体本身的旋转。

Unity中的Inspector窗口的Rotation就是欧拉角，脚本获取为

同一旋转在同一坐标系下的欧拉角表示不唯一（多旋转一周），且会出现万向节死锁

万向节死锁（Gimbal Lock），指在旋转过程中某两个旋转轴重合后，会导致失去一维自由度。因为三个旋转轴的旋转序列有先后顺序，当物体绕轴旋转时，上一层级的轴旋转时，会带动下面层级的轴旋转，而下一层级轴的旋转不会影响上面层级，则序列中间的那个轴在旋转时，会导致上一层级的轴和下一层级的旋转轴重合（Unity为X轴旋转90度产生万向节死锁）。

使用欧拉角旋转时，可以转换为矩阵旋转（具体每个轴的旋转矩阵见Games101笔记），而每次的变换是从初始姿态开始算起的，而不是从中间的某个状态开始的（只是可以用三个矩阵进行表示）；欧拉角只负责变换，不负责记录用户操作；欧拉角和运动没有任何关系，它描述的是变换！因此，三个旋转矩阵在第一个矩阵左乘后，后续矩阵的旋转参考系没有更新（在不更新坐标轴的情况下瞬间进行三个方向的旋转），导致三个矩阵简化后两个自由度的旋转坍缩为一个自由度。

再次回到Unity的万向节死锁上，会发现X轴旋转90度后，此时旋转的Y轴和Z轴不在同一个坐标系，即绕Z轴为绕物体本身的坐标轴，而绕Y轴为绕世界坐标值。

四元数定义

一个四元数包含一个标量和一个3D向量，q=w+xi+yj+zk=[w,(x,y,z)]，单位四元数满足 w2+x2+y2+z2=1。注意上面 q 的表示公式中的 x,y,z 并不是旋转轴的坐标，而是四元数的虚部。给定旋转轴 \boldu（单位向量）和旋转角度 θ，对应的四元数为

q=(cos⁡(θ2),\boldusin⁡(θ2))

假设有一个向量 \boldv=(vx,vy,vz)，将其绕轴 \boldu 旋转角度 θ。为了用四元数表示旋转，需要将向量 \boldv 表示为纯四元数，\boldv=(0,\boldv)，旋转后的向量 \boldv′=q\boldvq−1，其中 q=(cos⁡(θ2),−\boldusin⁡(θ2))。

旋转操作应该保持向量的长度不变。如果只计算 q\boldv，结果的长度可能会发生变化通过引入 q−1，可以确保旋转后的向量 \boldv′ 的长度与原始向量 \boldv 相同。

四元数的乘法定义为

(a,\boldb)(c,\boldd)=(ac−\boldb⋅\boldd,a\boldd+c\boldb+\boldb×\boldd)

将对向量\boldv的旋转公式展开，将会得到Rodrigues旋转公式（见Games101笔记）

图片导入相关

纹理类型

  • Default：默认纹理，用于大部分模型贴图。
    • sRGB用于伽马矫正
    • Alpha Source：生成纹理Alpha通道的模式。None无论导入纹理是否有Alpha通道，导入后的纹理都没有Alpha通道；Input Texture Alpha按照导入纹理的Alpha通道
    • Alpha Is Transparency：用alpha来表示透明的纹理时需要勾选。勾上以后，引擎会把边界处透明的颜色设置成和临近的不透明区域相近的值，这样边缘的不透明像素在Texture Filtering或计算mipmap时，就不会和黑色混合了（透明=黑色），从而避免产生黑边。
  • Normal map：法线贴图
    • Create From Grayscale：启用后从灰度高度贴图创建法线贴图
    • Bumpiness：控制凹凸程度，值越大凹凸感越强
    • Filtering：计算凹凸值。Smooth使用标准算法；Sharp生成比标准模式更锐利的法线贴图
  • Sprite：用于2D或UGUI
    • Sprite Mode：提取精灵图像方式。Single单张图；Multiple图集，可进一步使用编辑器切分图片；Pilygon自定义多边形网格
    • Pixels Per Unit：世界坐标的一个单位对应多少个像素
    • MeshType：网格类型。Full Rect始终以矩形网格显示图片（类似AABB）；Tight根据图片的Alpha通道生成对应的网格（直接用三角面拼出图像的非透明部分）。任何小于32x32的图片即使选择Tight也会以Full Rect显示
  • Cursor：用于自定义光标的图案
  • Cookie：光影剪影格式
    • Spotlight：聚光灯类型，需要边缘纯黑色纹理
    • Directional：方向光，平铺纹理
    • Point：点光源

DrawCall

CPU准备好渲染数据（顶点、纹理、Shader等）后，告知GPU开始渲染的命令。较多的DrawCall会影响CPU效率，影响渲染效率的主要因素使CPU提交命令的速度

TileMap

Isometric Z as Y类型的TileMap在使用时，需要在Project Settings->Graphics->Transparency Sort Mode修改为Custom Axis，同时将Transparency Sort Mode修改为x:0 y:1 z:-0.26，最后将Tile Renderer组件上的的Mode改为Individual。

Tilemap Renderer

Sort Order：设置所选瓦片地图上的瓦片排序方向，当瓦片模式为Isometric时，可能需要选择Top Left

Mode：渲染器的渲染模式

  • Chunk：按位置对瓦片进行分组，并将瓦片一起批处理渲染，性能较好
  • Individual：单独渲染每个瓦片，会考虑位置和排序顺序

Tilemap Collider 2D组件，生成碰撞器的时候会按照创建瓦片时，选择的Collider Type类型进行生成，None表示不会生成，Grid表示按照格子大小生成，Sprite按照图片大小生成（图片不透明的位置）

Isometric等距瓦片地图使用事项

轴心点排序：Sprite Renderer中的Sprite Sort Point默认为Center表示以图片的中心点位置判断前后关系，改为Pivot后按照图片导入时设置的轴心点判断和其他图片的前后关系。需要注意，这种用法在创建会与玩家发生前后遮挡的TileMap时（场景中的树等），需要修改TileMap的Mode为Individual，且排序层与玩家层一致。

Collider Type：地图的碰撞器可直接使用Tilemap Collider 2D，但需要注意瓦片的Collider Type需要改为Grid

反向动力学IK

IK Solvers：IK解算器，添加IK时有三个参数，注意下面的“能否反向”，是指父节点是否旋转到反向

  • Chain(CCD)：循环坐标下降法，可以自定义影响N个关节点，从末端关节开始，逐关节旋转，逐步逼近目标位置。可能出现局部最优解而导致扭曲
  • Chain(FABRIK)：前后向到达IK，可以自定义影响N个关节点，先从末端到根节点调整位置，再从根节点重新连接骨骼。全局优化使结果更平滑自然，但性能开销更大
  • Limb：只会影响3个关节点，模拟人的肢体，只能单向弯曲

Effector：决定IK节点的默认位置，一般在对应的骨骼节点末端新建空对象，当reset pose时，会将Target重置到这个位置

Chain Length：决定能够控制多少个节点

Target：根据默认IK节点生成的真正IK节点对象，IK计算时按照这个节点位置计算

Flip：Limb模式的参数，决定弯曲方向

换装相关

2022LTS后的版本不能直接在Sprite Editor中直接修改Category，需要在Project中新建2D->Sprite Library Asset，再手动将各个替换组件进行分组标记。在根对象上添加Sprite Library组件并关联Sprite Library Asset，再给需要换装的子对象上添加Sprite Resolver组件。注意如果导入PSD文件时导入隐藏层，此时直接将文件拖入Hierachy窗口，需要手动删除重复的子对象，每个换装部位仅留一个子对象即可，且保留的这个子对象应是绑定了骨骼的。

_SkeletonData参数
