垃圾回收机制

垃圾回收（Garbage Collector）只负责堆（Heap）内存的回收，引用类型都是存在堆中的，其分配和释放都通过垃圾回收机制进行管理。栈（Stack）上的内存是由系统自动管理的，值类型在栈上分配内存，有自己的声明周期，不用对其进行管理，会自动分配和释放。

引用类型声明时，会先在0代内存上进行分配内存，当0代内存没有多余空间时就会触发垃圾回收机制。

  1. 检测0代内存中的对象，将没有引用的对象进行释放，将有引用的对象移动到1代内存，并且使得1代内存地址连续（搬迁压缩）；
  2. 当1代内存也没有多余空间时，会触发1代内存的垃圾回收机制，将0代和1代内存中没有引用的对象都释放掉，并将1代内存中有引用的对象移动到2代内存；
  3. 当2代内存也没有多于内存时，会触发2代内存的垃圾回收机制，将0代、1代和2代中没有引用的对象都释放掉。
  4. 需要注意的是，为了减少性能损耗，调高性能，大对象总被储存在2代内存中，且不会被搬迁压缩

手动执行

可以在加载进度条等位置，手动触发垃圾回收机制。通常不会频繁调用，过大的性能消耗会导致一定的卡顿。

继承

子类只能有一个父类，子类间接继承父类的父类。

里氏替换原则

子类必须能够替代其父类被使用，即在软件设计中，程序中任何使用父类的地方，都可以透明地使用子类对象，而不会产生错误或意外行为。行为一致性，即子类在扩展父类功能的同时，不能破坏父类原有的行为约定，因此子类不能随意更改父类的预期行为。

总的来说即使用父类引用装载子类对象。

如果使用父引用装载子类对象，在调用子类扩展的方法时，需要用as转换或者强制转换后再调用。因为引用类型决定了可以访问的成员范围，父类引用只能访问父类中定义的成员，而无法直接访问子类中特有的成员或扩展方法。

方法重写（Override）和方法隐藏

  1. 方法重写

    • 父类的方法必须使用 virtual 或 abstract 修饰符。
    • 子类必须显式使用 override 关键字来重写父类方法。
    • 方法重写会支持运行时多态（动态绑定），即方法的调用依赖于对象的实际类型，而不是变量的编译时类型。

  2. 方法隐藏

    • 如果父类的方法没有 virtual 修饰，子类中定义了一个同名方法（无论是否显式加 new），子类方法会隐藏父类的方法，而不是重写。
    • 在这种情况下，方法调用会基于变量的编译时类型（即引用类型），而不是运行时类型。

关于父类的无参构造函数

子类的构造函数在实例化时，会默认调用父类的无参构造函数，若父类声明了其他构造函数且没有显式声明无参构造函数，则会发生报错。

但可以使用base关键字去指定调用的构造函数来避免报错。

关于关键字virtual和abstract的区别

特性 virtual abstract
是否可以有实现 可以有实现，提供默认行为。 不可以有实现（没有函数体 ），子类必须实现。
子类是否必须重写 子类可以选择重写，也可以使用父类实现。 子类必须重写，否则编译错误。（意味着不能private）
父类是否可以实例化 可以实例化（普通类）。 不可以实例化（抽象类）。
使用场景 提供默认实现，允许子类自定义行为。 强制所有子类实现该方法的行为。

抽象类虽然不能被实例化，但仍然能够作为父类引用装载子类对象。

抽象类可以有非抽象方法，但抽象方法只能存在于抽象类中。

interface，只包含属性、方法、索引器和事件，访问修饰符不能是private。接口不能继承类，但能继承另一个接口（且可以继承多个接口），接口不能实例化，但能够作为父类引用装载子类对象。

事件

事件只能作为成员存在于类、接口或结构体中，不能作为临时遍历出现在方法中。不能在类的外部进行赋值（=）或调用，但可以添加（+=）或移除（-=）装载的方法

事件的意义是，防止外部随意将委托置空或调用委托，使得委托更加安全
